# NIS2-CZ ruleset milestones (core pilot)

Status: complete

## Definition of done (pilot)
- Covers the **core CZ KB package** (Act 264/2025 + decrees 408/409/410/334)
- Includes a mapping reference to NIS2 Directive where useful
- Each rule has: id, title, scope/conditions, requirement, evidence_hints.required, severity, sources
- Split into small milestones that fit within scheduled runs (10–40 min) with lock

## Milestones

0) Foundations
0.1) Rule conventions + evidence artifact taxonomy + scoping approach
0.2) Applicability/scoping rules (in-scope + essential/important + higher/lower)

1) Governance & accountability
1.1) Management responsibility + policy set
1.2) Roles, training, awareness, internal audit

2) Risk management measures (CZ measures derived from decrees)
2.1) Asset mgmt + classification
2.2) Access control + identity + logging
2.3) Vulnerability mgmt + patching
2.4) Backup + BCP/DR + resilience
2.5) Secure development / change management

3) Incident handling & reporting (CZ portal / workflows)
3.1) Incident classification + register
3.2) Reporting workflow + timelines + content (CZ)
3.3) Post-incident review + improvements

4) Supply chain / third-party
4.1) Supplier register + risk assessment
4.2) Contract clauses + monitoring
4.3) Exit/transition planning

5) Finalization
5.1) Consistency sweep + generate completion note
