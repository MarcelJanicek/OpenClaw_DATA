# DORA ruleset milestones

Status: in progress

## Target definition of done
- Covers DORA obligations end-to-end with practical compliance-check focus
- Parameterized by `rules/dora/entity_profile.schema.yaml`
- Each rule includes:
  - id, title
  - scope (assessed party type, applicability conditions)
  - requirement (test type + acceptance criteria)
  - evidence_hints.required (explicit list)
  - severity
  - sources (DORA chapter/article citations)

## Milestones (20–40 min slices)

0) Foundations
0.1) Scope + definitions + applicability model (set rule ID conventions, common evidence artifacts)

1) ICT risk management framework (core governance)
1.1) Governance & responsibility model
1.2) Policies/standards: risk management, classification, protection
1.3) Detection, response & recovery; backup; crisis comms
1.4) Testing requirements (baseline) + remediation tracking

2) ICT-related incident management, classification & reporting
2.1) Incident classification process + register
2.2) Reporting workflow + timelines + content requirements
2.3) Post-incident review & lessons learned

3) Digital operational resilience testing (advanced)
3.1) Testing program governance
3.2) TLPT readiness (if applicable) + evidence

4) ICT third-party risk management
4.1) Register of information + vendor inventory
4.2) Pre-contract due diligence + risk assessment
4.3) Contract clauses requirements (outsourcing agreements)
4.4) Ongoing monitoring + concentration risk
4.5) Exit strategies + termination/transition planning

5) Information sharing arrangements
5.1) Threat intel sharing governance + confidentiality

6) Oversight / supervisory cooperation (tag + evidence)
6.1) Competent authority interactions; documentation readiness

7) Final pass
7.1) Consistency sweep (required evidence lists, naming, severity) + generate completion note
